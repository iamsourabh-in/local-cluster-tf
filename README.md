# Local Kubernetes Platform (Kind + Prometheus + Loki + Jaeger + OTel + Cert-Manager)

This repository contains a Terraform configuration to spin up a local Kubernetes cluster using `kind` (Kubernetes in Docker) and automatically deploy a full platform stack.

## Components Installed
- **Kind Cluster**: Exposed on host ports 80 (HTTP) and 443 (HTTPS) for Ingress.
- **Ingress Nginx**: Ingress controller routing local HTTP/HTTPS requests.
- **Metrics Server**: Collects resource metrics (`kubectl top nodes/pods`).
- **Cert-Manager**: Manages local certificates, pre-configured with a self-signed `ClusterIssuer` (`selfsigned-issuer`).
- **Kube-Prometheus-Stack**: Deploys Prometheus, Grafana, and basic node exporters.
- **Loki**: Log aggregation server running in Single-Binary mode with filesystem storage.
- **Promtail**: Log agent that ships stdout/stderr container logs to Loki.
- **Jaeger**: Trace visualizer and collector running in All-in-One mode.
- **OpenTelemetry Collector**: Receives OTLP metrics, traces, and logs, routing them to Prometheus, Jaeger, and Loki respectively.

## Prerequisites
- **Docker**: Must be running on your system.
- **Terraform** (>= 1.5.0)
- **Kubectl** (to interact with the cluster)

---

## Getting Started

### 1. Provision the Cluster & Platform
Run the following commands to initialize and apply the Terraform configuration:
```bash
cd platform
terraform init
terraform apply -auto-approve
```
This will take a few minutes as it spins up the Kind node, fetches Helm repositories, and provisions all services.

### 2. Configure Local DNS
Once the deployment finishes, add the local host mappings to your `/etc/hosts` file:
```text
127.0.0.1 grafana.local prometheus.local jaeger.local example-app.local
```

### 3. Verification & Access
Now, access your platform dashboards:
* **Grafana**: [http://grafana.local](http://grafana.local) (Username: `admin` | Password: `prom-operator`)
  * Loki and Jaeger are pre-configured as datasources in Grafana!
* **Jaeger**: [http://jaeger.local](http://jaeger.local)
* **Prometheus**: [http://prometheus.local](http://prometheus.local)

## Deploying an Application (Example)

We have provided a demo application (`example-app.yaml`) that is pre-instrumented with OpenTelemetry to export traces, prints structured logs (collected by Promtail), and uses TLS certificates generated by Cert-Manager.

### 1. Deploy the Application
```bash
kubectl apply -f example-app.yaml
```

Wait for the deployment to become ready:
```bash
kubectl rollout status deployment/example-app -n example-app
```

### 2. Generate Traffic
Access the application in your browser at [https://example-app.local](https://example-app.local) (accept the self-signed certificate warning), or run:
```bash
curl -k https://example-app.local/
```

### 3. View Observability Data
- **Traces in Jaeger**: Open [http://jaeger.local](http://jaeger.local) and select service `example-app` to view the distributed traces (including nested `simulated-db-query` and `external-api-call` spans).
- **Logs in Loki**: Open Grafana, go to **Explore**, select the **Loki** datasource, and run the query:
  ```text
  {namespace="example-app", container="python-app"}
  ```
- **Unified Traces-to-Logs**:
  Within Grafana's Jaeger trace view, click on any span and you will see a **"Document"** link next to the span details that links directly to the corresponding Loki container logs around that span's timestamp!

## Cleanup
To destroy the local cluster and all associated resources:
```bash
terraform destroy -auto-approve
```
